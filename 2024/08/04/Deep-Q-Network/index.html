Value Function Approximation

In Q learning, we try to store all the Q values in a Q table,which is a matrix. However, this method of storing Q value is only suitable environment and action spaces which are both small and discrete. We the action space or state space is continuous, which means we probably have countless number of action and state, then we can not use a Q table to store those data.

In that, We can try to use value function approximation (VFA) to anticipate and fit Q value. VFA is a method used in reinforcement learning to estimate the value of states or state-action pairs. This is particularly useful when dealing with large or continuous state spaces where it is impractical to maintain exact values for all possible states or state-action pairs. There are many methods to conduct VFA, such as linear function and non-linear function (neural networks).

Artificial neural networks (ANNs)

Artificial Neural Networks (ANNs) are computational models inspired by the human brain, designed to recognize patterns and solve complex problems. They consist of layers of interconnected nodes (neurons) that work together to process input data and generate output.

Neurons and layers

Neurons are the basic unit of a neural network that receives input, processes it, and produces an output.

An ANN usually has three layers: input layer, hidden layer and output layer. The input layer receives the initial data, the hidden layers are intermediate layers that perform computations and feature extraction, the output layer produces the final output.

Weights and Biases

Weights are parameters that determine the strength of connections between neurons, while biases are parameters added to the weighted sum of inputs to help the model learn more complex patterns.

Activation Functions

Activation functions are non-linear functions applied to the output of each neuron. Common activation functions include:

Sigmoid
$$
\sigma(x) = \frac{1}{1 + e^{-x}}
\tag{1-1}
$$

ReLU
$$
ReLU(x)=max⁡(0,x)
\tag{1-2}
$$

tanh
$$
tanh(x) = \frac{e^x - e^{-x}}{e^x + e^{-x}}
\tag{1-3}
$$

Neural network training

We have several different methods to train the neural networks, make it works like a human brain.

  1. forward propagation:

    input data is passed through the network layer by layer, and each neuron processes the input, applies the activation function, and passes the result to the next layer.

  2. Loss function:

    Use loss function to measure the difference between the network’s output and the true target values.

  3. Backward propagation :

    Algorithms use backward propagation to adjust weights and biases to minimize the loss. Involves calculating the gradient of the loss function with respect to each weight and bias using the chain rule. Gradients are used to update the parameters via optimization algorithms like Gradient Descent. The amplitude of variation in parameters’ update determined by learning rate $\alpha$ of the neural networks.

DQN = Deep network + Q learning

Algorithm

We can use neural network to process Q value. If the action space and state are both continuous, we can input action and state into neural network and output a value as Q value; if action space is discrete, we can input state and output a Q value. Since Q learning have to find the max Q value of each state, the DQN can only handle discrete action spaces.

The update rule of Q learning is as below:
$$
Q(s,a) \leftarrow Q(s,a)+\alpha [r+\gamma \mathop{\max}\limits_{a’\in A}Q(s’,a’)-Q(s,a)] \tag{2}
$$
This update rule is based on Temporal Difference (TD) to make $Q(s,a)$ approximate to $r+\gamma \mathop{\max}\limits_{a’\in A}Q(s’,a’)$. Thus, we can define the loss function of DQN as:
$$
\omega ^*=\mathop{\arg}{\min}\limits_{\omega} \frac{1}{2N}\sum_{i=1}^{N}[Q_{\omega}(s_i,a_i)-(r_i+\gamma \mathop{\max}\limits_{a’}Q_{\omega}(s’_i,a’))]
\tag{3}
$$

Target network

Since the TD error target itself contains the output of the neural network, the target is constantly changing while updating the network parameters, which is very easy to cause instability in the training of the neural network. In order to solve this problem, DQN uses the idea of target network: since the constant updating of the Q network during the training process will cause the target to keep changing, it is better to fix the Q network in the TD target for the time being. In order to realize this idea, we need to use two sets of Q networks:

  1. The Q network still calculates $Q(s,a)$, and update through gradient descent;
  2. The target Q network calculates $r+\gamma \mathop{\max}\limits_{a’\in A}Q(s’,a’)$. To make target updating more stable, the target network is not updated every step. Specifically, the target network uses an older set of parameters from the training network, which is updated at every step in training, while the target network’s parameters are only synchronized with the training network every $n$ steps.

Experience replay

In general supervised learning, assuming that the training data are independently and identically distributed, we randomly sample one or more pieces of data from the training data each time we train the neural network to perform gradient descent, and each piece of training data is used multiple times as learning continues. In the original Q learning algorithm, each piece of data is used only once to update the Q value. In order to better integrate Q learning and deep neural networks, the DQN algorithm uses an experience replay method, which consists of maintaining a replay buffer, storing a quaternion of data (state, action, reward, and next state) each time the environment is sampled, and then randomly sampling a number of data from the replay buffer when training the Q network. buffer, and when training the Q network, randomly sample a number of data from the playback buffer for training. Doing so serves two purposes:

  1. Make the samples satisfy the independence assumption. In MDP, the data obtained from cross-sampling does not satisfy the independence assumption by itself, because the state at this moment is related to the state at the previous moment. Data that are not independently and identically distributed have a significant impact on training the neural network, causing the neural network to fit to the most recently trained data. Using empirical playback can break the correlation between samples and allow them to satisfy the independence assumption.
  2. Improve sample efficiency. Each sample can be used multiple times, very suitable for gradient learning of deep neural networks.

Implementation

These are the necessary modules the implementation needs:

Q network

We classify Q network. First, we initialize function and receive two parameters: state and action. Secondly, we build up the neural network structure:

  1. Define a fully connected layer fc1 that transforms the input state into a 64-dimensional hidden layer. nn.Linear denotes a linear transformation, i.e., a fully connected layer.
  2. Define a second fully-connected layer, fc2, that converts the output of the second hidden layer into an action-sized output. This is the output layer that is used to generate the Q value for each action.
  3. Define the forward propagation function forward, which accepts the state state as input.
  4. The input state state is passed to the first fully connected layer fc1, which then activates the function via ReLU.
  5. Pass the output of the second hidden layer to the second fully connected layer fc2 and return the final output. This is the Q value for each action.

Replay buffer

The replay buffer class is very useful in deep reinforcement learning, allowing algorithms to train by replaying past experience data, thereby improving sample efficiency and breaking the temporal correlation of the data.

self.buffer uses the collections.deque data structure to store experience data. deque is a double-ended queue that supports efficient insertion and deletion operations. The maxlen=capacity parameter sets the maximum length of the queue. When the queue reaches its maximum length, the oldest data is automatically removed if new data is added;

The add method is used to append new experience data to the buffer. state, action, reward, next_state, and done are the experience data obtained from the environment at each step. These data are added to the buffer as a tuple;

The sample method is used to randomly sample a batch of data from the buffer for training the model. The batch_size parameter specifies the number of samples. random.sample(self.buffer, batch_size) randomly selects batch_size experience data from the buffer. zip(*transitions) unzips these data, separating them into states, actions, rewards, next states, and done flags. np.array converts states and next states into NumPy arrays for numerical computation. Finally, the method returns the states, actions, rewards, next states, and done flags;

size returns the number of data currently in the buffer.

Agent

After the build-up of replay buffer and neural networks, we can start to build the DQN agent. state_dim is the dimension of state, action_dim is the dimension of action, hidden_dim is the dimension of hidden layer.

The Q network (q_net) which approximates the action-value function, and the target Q network is used to stabilize training by providing consistent targets for the Q-value updates.

This is $\epsilon-greedy$ algorithm which encourage agent to explore at beginning and guide agent to take action which can bring maximum Q value after certain iterations.

The update method in the DQN class performs the core operation of updating the Q-network based on a batch of transitions.

  1. Convert the transition data (states, actions, rewards, next_states, and dones) from the transition dictionary to PyTorch tensors. The data are moved to the specified device (CPU or GPU). view(-1, 1) reshapes the tensors to ensure proper dimensions for the calculations.
  2. Compute the Q-values for the current states using the Q network. gather(1, actions) selects the Q-values corresponding to the taken actions.
  3. Compute the Q-values for the next states using the target Q-network. max(1)[0] selects the maximum Q-value for each next state, representing the best possible action. view(-1, 1) reshapes the tensor for proper dimensions.
  4. Calculate the target Q-values using the Bellman equation.(1 - dones) ensures that if the episode is done (done is 1), the future reward is not considered.
  5. Calculate the mean squared error (MSE) loss between the predicted Q-values (q_values) and the target Q-values (q_targets).
  6. Conduct backpropagation and optimizer step.zero_grad() clears the gradients of all optimized tensors.backward() computes the gradient of the loss with respect to the network parameters.step() updates the network parameters using the optimizer.
  7. Update target network. In every target_update steps, updates the target Q-network with the current Q-network’s weights to ensure stable targets during training and increment the step counter self.count.

Environment and hyperparameters

Environment of this DQN implementation is classic cartpole-v0. The action is discrete: go left or go right, the state is the angle formed between the rod on the cart and the vertical line.

Result

We can see the learning efficiency of DQN improves rapidly after 100 episodes and the optimal returns converges to 200. The existence of oscillation is due to the effect of argmax operation after the neural network is overfitted to some local empirical data.
